# ZoneForge — Claude Code Context

## Project Summary

ZoneForge is a multiplayer-ready isometric RPG world editor. It consists of three applications sharing a single SpacetimeDB backend: a standalone world editor, a game client, and a Rust server module. All three are separate git submodules under this umbrella repo.

| Client↔Server | SpacetimeDB C# SDK |

## Key Conventions

**Server module**
- Entry point: `server/spacetimedb/src/lib.rs` — all tables and reducers live here (split into modules as it grows)
- Always use `spacetime build`, not `cargo build` — the CLI handles the WASM target
- Use `spacetime publish --server local zoneforge-server` to deploy locally
- Add `--delete-data` only for breaking schema changes (wipes all table data)
- `#[table(accessor = ...)]` on structs — do NOT also add `#[derive(SpacetimeType)]` to table structs
- `#[derive(SpacetimeType)]` is only for custom types embedded as fields within rows

**Client bindings (autogen)**
- Generated by: `spacetime generate --lang csharp --out-dir Assets/Scripts/autogen --bin-path ../server/spacetimedb/target/wasm32-unknown-unknown/release/zoneforge_server.wasm`
- Run from the `client/` or `editor/` directory after any server schema change
- `Assets/Scripts/autogen/` is generated — never edit manually

**Database name:** `zoneforge-server` (lowercase with dashes — SpacetimeDB requirement)

## Daily Dev Loop

```bash
# Terminal 1 — keep running
spacetime start

# Terminal 2 — after server changes
cd server && spacetime build && spacetime publish --server local zoneforge-server

# Regenerate bindings if schema changed (run from client/ or editor/)
spacetime generate --lang csharp --out-dir Assets/Scripts/autogen \
  --bin-path ../server/spacetimedb/target/wasm32-unknown-unknown/release/zoneforge_server.wasm
```

## Important Gotchas

- `wasm-opt` warning during `spacetime build` is non-critical — ignore during development
- Unity auto-generates `.vscode/settings.json` — do not overwrite it
- The editor project (`editor/`) is a **standalone runtime app** — no `EditorWindow`, `AssetDatabase`, or other Editor-only APIs
- `spacetime sql` and `spacetime call` print an UNSTABLE warning — that's expected, not an error

## Submodule CLAUDE.md Files

Each submodule carries its own CLAUDE.md with context specific to that codebase:
- `server/CLAUDE.md` — SpacetimeDB Rust SDK rules, common LLM mistakes, reducer/table patterns
- `client/CLAUDE.md` — Unity C# conventions, SpacetimeDB C# SDK patterns, autogen workflow
- `editor/CLAUDE.md` — Standalone editor app conventions, runtime UI patterns, 3D tile grid

Claude Code loads both the root CLAUDE.md and the submodule CLAUDE.md when working inside a submodule directory.
