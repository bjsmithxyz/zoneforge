## SpacetimeDB Integration

The client communicates with the server exclusively through the **SpacetimeDB C# SDK**.

**Connection lifecycle** (managed by `SpacetimeDBManager`):
1. `DbConnection.Builder()` вЂ” connect to `http://localhost:3000` (dev) or cloud URL (prod)
2. `OnConnect` вЂ” subscribe to relevant tables
3. `OnSubscriptionApplied` вЂ” register table event callbacks (`OnInsert`, `OnUpdate`)
4. `FrameTick()` вЂ” called every `Update()` to process incoming messages

**Generated bindings** вЂ” `spacetime generate --lang csharp` produces typed C# classes for every table and reducer in the server module. Output goes to `Assets/Scripts/autogen/`. Regenerate after any server schema change.

## Editor Tools

Custom Unity Editor Windows provide the world-building interface:

| Window | Menu Path | Purpose |
|--------|-----------|---------|
| `MapEditorWindow` | ZoneForge в†’ Map Editor | Tile painting and zone layout |
| `ZoneEditorWindow` | Tools в†’ ZoneForge в†’ Zone Editor | Zone creation with SpacetimeDB registration |

Editor scripts live in `Assets/Scripts/Editor/` and are excluded from game builds by Unity's compiler.

## Asset Pipeline

Textures are authored procedurally and painted using external tools:

- **Material Maker** вЂ” node-based procedural texture graphs (`.mmg` в†’ albedo/normal/roughness PNG sets)
- **ArmorPaint** вЂ” 3D texture painting with smart materials (edge wear, dirt, ambient occlusion)
- **Blender** вЂ” 3D modelling and Geometry Nodes for procedural foliage/scatter

All exported maps land in `Assets/Art/Materials/Exports/` and are referenced by Unity Materials.

## Object Pooling

Combat projectiles and VFX use `ZoneForgePoolManager` to avoid runtime allocations. Components `PooledProjectile` and `PooledVFX` must be present on all combat prefabs. Pool capacities are tuned during load testing (В§8A in Detailed Design).
